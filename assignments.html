4 Assignments + Final Project

Course Assignments

Hands-on projects designed to build your skills progressively. Each assignment challenges you to apply course concepts to real development scenarios.

All assignments must be submitted via Gradescope by 11:59 PM on the due date. Late submissions receive a 10% penalty per day (max 3 days).
A1
Setup & Basic Prompting
Due: Week 2 — Jan 31, 2026
10 points

Get your development environment configured and practice fundamental prompting techniques. This assignment establishes the foundation for all future work in the course.

Learning Objectives:
  • Install and configure Claude Code in VS Code or terminal
  • Write effective zero-shot and few-shot prompts
  • Understand model limitations and context constraints
  • Document your prompting strategies and outcomes
Tasks:
  1. Environment Setup (2 pts): Install Claude Code, configure API keys, verify functionality with a "Hello World" script
  2. Prompt Experiments (4 pts): Generate code for 3 different tasks (data processing, web scraping, algorithm implementation) using various prompting strategies
  3. Comparative Analysis (2 pts): Compare zero-shot vs few-shot results for the same task, documenting quality differences
  4. Reflection (2 pts): Write 500 words on what makes prompts effective based on your experiments
Submission Format:
  • GitHub repository with code, prompts, and outputs
  • README.md with setup instructions and reflection
  • prompts/ folder containing all prompts used (text files)
  • outputs/ folder with generated code and results
A2
Context Engineering Challenge
Due: Week 5 — Feb 21, 2026
10 points

Work with a medium-sized codebase (provided) and demonstrate mastery of context management. Learn when and how to provide relevant information to the model.

Learning Objectives:
  • Navigate and understand unfamiliar codebases efficiently
  • Optimize context windows for complex tasks
  • Use @-mentions and file references effectively
  • Balance between too little and too much context
Tasks:
  1. Codebase Analysis (3 pts): Clone the provided repository (a Flask web app), understand its structure, and document key components
  2. Feature Implementation (4 pts): Add a new feature (user authentication) by providing optimal context to Claude. Track which context you included and why
  3. Context Optimization (2 pts): Demonstrate the same task with minimal vs maximal context. Compare results and token usage
  4. Documentation (1 pt): Create a guide on context selection strategies for this codebase
Submission Format:
  • Fork of the provided repository with your feature branch
  • CONTEXT_STRATEGY.md documenting your approach
  • Conversation logs showing context decisions (exported from Claude)
  • Pull request with detailed description and rationale
A3
Build a Feature with Agentic Workflow
Due: Week 9 — Mar 21, 2026
10 points

Design and implement a complete feature using agentic workflows. Claude should autonomously perform multiple steps including file operations, testing, and iteration.

Learning Objectives:
  • Design multi-step autonomous workflows
  • Enable and verify tool use (file operations, shell commands)
  • Handle errors and implement recovery strategies
  • Maintain consistency across multi-file changes
Tasks:
  1. Feature Specification (2 pts): Define a feature that requires 5+ file changes and includes backend + frontend components (e.g., REST API endpoint with UI)
  2. Agentic Implementation (5 pts): Use Claude to autonomously create files, write code, generate tests, and verify functionality. Minimal human intervention
  3. Error Recovery (2 pts): Document at least one error encountered and how Claude recovered or how you guided recovery
  4. Workflow Diagram (1 pt): Visual flowchart showing the steps Claude took (tools used, decisions made)
Submission Format:
  • GitHub repository with complete feature implementation
  • WORKFLOW.md documenting the agentic process
  • Full conversation transcript (exported from Claude)
  • Workflow diagram (PNG or PDF)
  • Demo video (2-3 minutes) showing the feature in action
A4
AI-Assisted Code Review
Due: Week 12 — Apr 11, 2026
10 points

Conduct comprehensive code reviews using Claude. Analyze code quality, identify bugs, suggest improvements, and integrate AI into a CI/CD pipeline.

Learning Objectives:
  • Perform AI-assisted code review at scale
  • Integrate Claude into GitHub Actions workflows
  • Balance automated suggestions with human judgment
  • Generate actionable, constructive feedback
Tasks:
  1. Manual Review (3 pts): Review 3 pull requests from the provided repository. Use Claude to analyze code quality, security, performance, and style
  2. CI Integration (4 pts): Create a GitHub Action that runs Claude-based code review on every PR. Action should post review comments automatically
  3. Comparative Analysis (2 pts): Compare AI review with human review (instructor-provided). Identify false positives/negatives and discuss limitations
  4. Best Practices Guide (1 pt): Write guidelines for effective AI-assisted code review in team settings
Submission Format:
  • GitHub repository with custom GitHub Action workflow
  • Review comments posted on 3 PRs (links provided)
  • REVIEW_ANALYSIS.md comparing AI vs human review
  • CODE_REVIEW_GUIDE.md with team best practices
Final
End-to-End Development with Claude Code
Presentations: Week 14 — Apr 21-25 | Submission: May 12, 2026
40 points

Capstone project demonstrating comprehensive mastery of AI-assisted development. Build a complete, production-ready application using advanced Claude Code workflows from inception to deployment.

Learning Objectives:
  • Design and implement a complete application architecture
  • Apply advanced prompting, agentic workflows, and tool use
  • Integrate testing, CI/CD, and deployment automation
  • Reflect critically on AI assistance benefits and limitations
Project Requirements:
  • Scope: Full-stack application or complex backend system (min 1,000 lines of code)
  • Features: At least 3 major features, each requiring multi-step implementation
  • Testing: Comprehensive test suite (unit + integration tests, >80% coverage)
  • CI/CD: Automated testing and deployment pipeline
  • Documentation: README, API docs, and architectural decision records
  • Deployment: Live demo accessible via URL
Grading Breakdown:
  • Implementation Quality (15 pts): Code quality, architecture, functionality, and completeness
  • AI Workflow Sophistication (10 pts): Effective use of advanced prompting, agentic patterns, and tool integration
  • Testing & CI/CD (5 pts): Test coverage, automation, and deployment pipeline
  • Documentation (5 pts): Clear README, architectural docs, and conversation logs
  • Presentation (5 pts): Live demo, technical explanation, and reflection on AI assistance
Submission Format:
  • GitHub repository with complete project code
  • README.md with setup, features, and demo link
  • docs/ARCHITECTURE.md explaining design decisions
  • docs/AI_WORKFLOW.md documenting how Claude was used
  • Conversation logs (key interactions with Claude)
  • 15-minute presentation (slides + live demo)
Project Ideas (Examples):
  • AI-powered content management system with automated SEO optimization
  • Real-time collaborative code editor with AI-assisted refactoring
  • Developer productivity dashboard aggregating GitHub, Jira, and Slack data
  • Automated testing platform that generates test cases from user stories
  • Custom MCP server with domain-specific tools for your field

Important information

Submission Guidelines

Academic Integrity

You may use Claude Code for all assignments—that's the point! However, you must document your prompts and conversations. Submitting AI-generated code without understanding it violates academic integrity. Be prepared to explain your code in detail.

Late Policy

Late submissions receive a 10% penalty per day (max 3 days). After 3 days, assignments receive a zero. Extensions may be granted for documented emergencies—contact Prof. Chen at least 48 hours before the deadline.

Conversation Logs

All assignments require conversation logs showing your interaction with Claude. Export these as text files and include them in your submission. This helps us understand your thought process and troubleshooting strategies.

Resubmissions

Assignments 1-4 allow one resubmission within 7 days of grade release. Resubmissions can recover up to 70% of lost points. The Final Project does not allow resubmissions due to end-of-semester timing.
