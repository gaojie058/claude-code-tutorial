14 Weeks

Course Syllabus

A comprehensive 14-week journey through AI-assisted software development. From basic prompting to advanced agentic workflows and production deployment.

All readings should be completed before the week's first lecture. Labs are hands-on sessions—bring your laptop with Claude Code installed.
W1
Jan 20–24
Introduction to Claude Code & AI Development

Overview of AI-assisted development landscape. Introduction to Claude Code architecture, capabilities, and limitations. Course logistics and expectations.

Topics:
  • What is Claude Code? LLMs as coding assistants
  • Setting up development environment (VS Code, terminal integration)
  • First interactions: basic prompts and code generation
  • Understanding model context windows and limitations
Readings:
  • Anthropic Claude Code Documentation (Introduction & Getting Started)
  • "Language Models are Few-Shot Learners" (Brown et al., 2020) — excerpts
Setup Assignment
W2
Jan 27–31
Prompt Engineering Fundamentals

Learn the art and science of crafting effective prompts. Understanding how to communicate intent clearly, provide context, and iterate on prompts for better results.

Topics:
  • Anatomy of an effective prompt: clarity, context, constraints
  • Zero-shot vs few-shot prompting strategies
  • System prompts vs user messages
  • Debugging bad outputs through prompt refinement
Readings:
  • Anthropic Prompt Engineering Guide
  • "Chain-of-Thought Prompting Elicits Reasoning" (Wei et al., 2022)
Assignment 1 Due
W3
Feb 3–7
Context Management & Codebase Navigation

Working with large codebases effectively. Techniques for providing relevant context without overwhelming the model. File organization and project structure awareness.

Topics:
  • Understanding context windows: what to include, what to omit
  • Using @-mentions for files, folders, and documentation
  • Semantic code search and symbol navigation
  • Context hierarchy: immediate vs peripheral information
Readings:
  • Claude Code: Working with Projects documentation
  • "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks" (Lewis et al., 2020)
W4
Feb 10–14
Test-Driven Development with AI

Integrating AI into TDD workflows. Writing tests first, using Claude to implement functionality, and ensuring code quality through automated testing.

Topics:
  • Writing test specifications for AI to implement
  • Generating unit tests from existing code
  • Test coverage analysis and gap identification
  • Debugging failing tests with AI assistance
Readings:
  • "Test-Driven Development with AI Pair Programmers" (industry case studies)
  • Jest/Pytest documentation (review testing frameworks)
Assignment 2 Due
W5
Feb 17–21
Agentic Workflows & Multi-Step Tasks

Designing workflows where AI takes multiple autonomous steps. Understanding tool use, file operations, and iterative refinement patterns.

Topics:
  • Agentic behavior: when AI makes decisions
  • Tool use patterns: read, write, execute, search
  • Multi-file editing and refactoring workflows
  • Handling errors and recovery strategies
Readings:
  • "ReAct: Synergizing Reasoning and Acting" (Yao et al., 2023)
  • Anthropic Tool Use documentation
Assignment 3 Due
W6
Feb 24–28
Multi-Turn Conversations & Memory Management

Managing long conversations effectively. Techniques for maintaining context across sessions, handling conversation history, and optimizing for token efficiency.

Topics:
  • Conversation design patterns for development tasks
  • Context pruning and summarization techniques
  • Session persistence and state management
  • When to start fresh vs continue a thread
Readings:
  • "Long-Form Conversation Design Patterns" (research paper)
  • Claude API: Messages and Conversation History
W7
Mar 3–7
Midterm Project Presentations

Students present their midterm projects: a complete feature built using Claude Code. Peer feedback and technical Q&A. Reflection on first half of course.

Topics:
  • Project demonstrations (10 minutes each)
  • Technical deep-dive: prompts, workflows, challenges
  • Best practices sharing and lessons learned
  • Course retrospective and feedback session
Midterm Presentations
W8
Mar 10–14
Advanced Prompt Patterns & MCP Tools

Deep dive into advanced prompting techniques. Introduction to Model Context Protocol (MCP) and custom tool creation for specialized workflows.

Topics:
  • Chain-of-thought and reasoning patterns
  • Meta-prompting: prompts that generate prompts
  • MCP architecture and protocol overview
  • Creating custom tools for domain-specific tasks
Readings:
  • Anthropic MCP Documentation
  • "Tree of Thoughts: Deliberate Problem Solving" (Yao et al., 2023)
W9
Mar 17–21
Custom Commands & Workflow Automation

Building custom commands and automated workflows. Integrating Claude into shell scripts, CI pipelines, and development automation tools.

Topics:
  • CLI automation with Claude API
  • Custom slash commands and shortcuts
  • Integrating AI into existing developer tools
  • Workflow templates and reusable patterns
Readings:
  • Claude API Reference
  • "Building Developer Tools with LLMs" (industry best practices)
Assignment 4 Due
W10
Mar 24–28
Testing & Debugging with AI

Advanced debugging techniques using AI. Root cause analysis, error interpretation, and systematic debugging workflows.

Topics:
  • Error message interpretation and diagnosis
  • Stack trace analysis with AI assistance
  • Systematic debugging: hypothesis generation and testing
  • Performance profiling and optimization suggestions
Readings:
  • "AI-Assisted Debugging: Current State and Future Directions"
  • Debugging best practices (language-specific guides)
W11
Mar 31–Apr 4
CI/CD Integration & DevOps

Integrating AI into continuous integration and deployment pipelines. Automated code review, test generation, and deployment verification.

Topics:
  • GitHub Actions integration with Claude API
  • Automated PR review and suggestions
  • Test generation in CI pipelines
  • Release notes and changelog generation
Readings:
  • GitHub Actions documentation
  • "Continuous Integration with AI Code Review" (case studies)
W12
Apr 7–11
Team Workflows & Code Review with AI

Collaborative development with AI tools. Best practices for team coordination, code review processes, and maintaining code quality across team members.

Topics:
  • AI-assisted code review workflows
  • Team prompt libraries and shared practices
  • Consistency in AI-generated code across team
  • Documentation generation and maintenance
Readings:
  • "Human-AI Collaboration in Software Teams" (research paper)
  • Google Code Review Best Practices
W13
Apr 14–18
Security, Ethics & Responsible AI Use

Critical examination of AI in software development. Security considerations, bias detection, ethical implications, and responsible deployment practices.

Topics:
  • Security vulnerabilities in AI-generated code
  • Data privacy and code confidentiality
  • Bias in code suggestions and algorithmic fairness
  • Attribution, licensing, and legal considerations
Readings:
  • "Security Risks of AI Code Generation" (OWASP guidelines)
  • Anthropic Responsible Scaling Policy
  • "Fairness and Accountability in AI Systems" (Mitchell et al.)
W14
Apr 21–25
Final Project Presentations

Capstone project presentations: end-to-end development with Claude Code. Students showcase their complete applications, workflows, and lessons learned.

Topics:
  • Final project demonstrations (15 minutes each)
  • Technical architecture and design decisions
  • Challenges faced and solutions discovered
  • Course reflection and future of AI-assisted development
Final Project Due

Assessment

Grading Breakdown

40%
Assignments (4 × 10%)

Weekly assignments covering setup, prompt engineering, agentic workflows, and code review. Each assignment builds on previous concepts and includes both implementation and reflection components.

40%
Final Project

End-to-end application development using Claude Code. Must demonstrate advanced prompting, multi-step workflows, testing, and deployment. Includes written documentation and live presentation.

10%
Midterm Presentation

Mid-semester project demonstrating mastery of foundational concepts. 10-minute presentation covering technical implementation, challenges, and lessons learned.

10%
Participation

Active participation in class discussions, lab sessions, and peer code reviews. Includes attendance, contribution to class knowledge sharing, and engagement with course materials.
